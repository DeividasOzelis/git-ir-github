Git

Git - populiariausia kodod versijvimo programa.

Git komandos

  • git init - sukuria gito repozitorija dabartiniame aplankale.
  • status - nurodo info apie lokalios gito repozitorijos statusa.
  • add - prideda nurodyta faila prie commit'inimui paruostu failu(staged).(add . - prideda viska esamoj direktorijoj ir nereikia nurodinet kiekvieno failo atskirai)
  • rm --cached - isima nurodytus failus is stage'inimo vietos.
  • commit -m "zinute" - sukuria kodo versija is steidzintu failu ir nurodo zinute kas buvo keista kode, kokia versija ar panasiai
  • restore - grazina nurodytus failus i buvusio komito busena.
  • push - is lokalios repozitorijos nusiuncia commit'us i isorine repozitorija
  • pull - is isorines repozitorijos atsiuncia i lokalia repozitorija
  • clone - clonuoja nurodyta repozitorija is isorines i lokaliagit
  • log - isveda commit'u sarasa
  • checkout + commitID/branchName - pereina i nurodyta kodo versija ARBA i kita branch'a
  • branch pavadinimas - naujo branch'o sukurimas
  • merge name - sujungia esama saka su nurodyta name

Branch protection rules

Viena svarbiausiu saku apsaugos taisykliu yra 'Pull requests'(bandymas vienos sakos commit'us sukelti i kita saka).
Pull request dar galima suteikti 'Pull approvals', kuris reikalauja, kad nurodytas kiekis kitu projekto dalyviu patvirtintu 'Pull request'a'.

Pull request eiliskumas

  1. Susikrti nauja saka. git branch name
  2. Nueiti i ta saka.git checkout branchName
  3. Atlikti savo uzduoti savo sakoje.
    1. Addini pakitusius failus. git add
    2. Commit'ini pakitimus.git commit -m "msg"
  4. Nueini i main/master saka. git checkout main/master
  5. Atnaujini main/master saka. git pull
  6. Grizti i nauja saka. git checkout newBranch
  7. Sujungi savo saka su main ir issprendi galimai atsiradusius konfliktus. git merge main
  8. Nusiunti naujos sakos pakeitimus i gtihuba. git push
  9. Sukurti githube Pull requesta
  10. Kiti komandos nariai patvirtina arba atmeta ta Pull requesta
